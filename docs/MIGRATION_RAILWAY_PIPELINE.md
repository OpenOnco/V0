# Migration: Weekly Scan → Railway-orchestrated Pipeline

**Goal:** stop running the weekly scan in a Claude session. Make `test-data-tracker` (Railway) the sole writer of all data artifacts. V0 becomes a pure consumer.

**Why:** the Claude-orchestrated scan hallucinates section completions and commit hashes. Underlying scripts work fine when actually invoked. Railway cron + exit codes can't confabulate.

## Phase 2 — Single orchestrator entry point

- [ ] Create `test-data-tracker/scheduler/weekly.js` that runs in this order:
  1. Crawl CMS / vendors / FDA / payers / NCCN → write `data/submissions/weekly-YYYY-MM-DD.json`
  2. Fetch PubMed papers for any new PMIDs in `evidence/meta/sources.json`
  3. `extract-claims` on new papers
  4. `verify-claims` cross-check
  5. `render-faq` → updated `physicianFAQ.js`
  6. Run `verify-pipeline-state.js` — exit non-zero on drift
  7. Build manifest of artifacts (paths + sha256)
- [ ] Each step exits non-zero on failure. No try/catch swallowing. No LLM-generated summary.
- [ ] Step output goes to structured logs, not prose.

**Smoke test:** run end-to-end on a copy with one fake PMID; confirm exit 0 and artifacts present.

## What never runs in Claude again

- Crawling
- PubMed fetch
- Claim extraction / verification (Claude API yes; Claude orchestrator no)
- FAQ rendering
- Anything that should produce a commit hash
